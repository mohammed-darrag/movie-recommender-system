\subsection*{1.1.3 Brief Comment on the Scatter Plots}
The scatter plots show how the data points are distributed. If the points form a straight-line pattern, the relationship is linear. If the points are scattered, curved, or have no pattern, then the relationship is weak or non-linear.

\subsection*{Pearson Correlation Coefficient}
The Pearson correlation coefficient $r$ ranges from $-1$ to $+1$:
\begin{itemize}
    \item $r = +1$ $\to$ perfect positive linear relationship
    \item $r = -1$ $\to$ perfect negative linear relationship
    \item $r = 0$ $\to$ no linear correlation
\end{itemize}
Values close to $\pm 1$ indicate strong linear dependence, while values near $0$ indicate weak or no linear relationship.

\section*{1.2.1 Design Matrix $X$}
The design matrix $X$ used in multiple linear regression is:
\[
X =
\left[
\begin{array}{cccc}
1 & x_1^{(1)} & x_2^{(1)} & \cdots \, x_D^{(1)} \\
1 & x_1^{(2)} & x_2^{(2)} & \cdots \, x_D^{(2)} \\
\vdots & \vdots & \vdots & \ddots \\
1 & x_1^{(N)} & x_2^{(N)} & \cdots \, x_D^{(N)}
\end{array}
\right]
\]
The entries are defined as:
\[
X_{ij} =
\begin{cases}
1, & \text{if } j = 0 \\[6pt]
x_j^{(i)}, & \text{for } j = 1, \ldots, D
\end{cases}
\]

\subsubsection*{A. Definition of the Design Matrix}
\begin{itemize}
    \item First column (bias term): \[
    X_{i0} = 1
    \]
    \item Remaining columns (feature values): \[
    X_{ij} = x_j^{(i)} \quad \text{for } j = 1, \ldots, D
    \]
\end{itemize}

\subsubsection*{B. Dimensions of the Design Matrix}
\begin{itemize}
    \item $N$ rows (samples)
    \item $D+1$ columns (bias + features)
\end{itemize}

\subsubsection*{C. Why the Loss Becomes Univariate When $D = 1$}
\begin{itemize}
    \item When $D = 1$, there is only one feature.
    \item Multiple regression reduces to univariate regression.
    \item Predictions and loss values are identical.
    \item Hence, $L_M = L_U$ when $D = 1$.
\end{itemize}

\subsection*{1.2.2 Verification of the Closed-form Implementation}
In this task, I constructed the design matrix $X$ by adding a bias column and placing all feature values in the remaining columns. Using this matrix, I applied the pseudoinverse to compute the closed-form solution for the model parameters. \subsection*{1.3 Polynomial Regression}
For a polynomial model of degree $K$, the design matrix $X$ is defined as:
\[
X =
\left[
\begin{array}{cccc}
1 & x^{(1)} & (x^{(1)})^{2} & \cdots \, (x^{(1)})^{K} \\
1 & x^{(2)} & (x^{(2)})^{2} & \cdots \, (x^{(2)})^{K} \\
\vdots & \vdots & \vdots & \ddots \\
1 & x^{(N)} & (x^{(N)})^{2} & \cdots \, (x^{(N)})^{K}
\end{array}
\right]
\]
In polynomial regression, we use only one feature $x$, but we allow the model to include higher–degree terms. A polynomial model of degree $K$ has the form:
\[
f_\theta(x) = b + w_1 x + w_2 x^2 + \cdots + w_K x^K
\]
To express this model in matrix form, we construct a design matrix $X$ where:
\[
X_{i0} = 1, \qquad
X_{ij} = (x^{(i)})^{j} \quad \text{for } j = 1, \ldots, K
\]
The dimensions of the matrix are:
\[
X \in \mathbb{R}^{N \times (K+1)}
\]
Using this design matrix, the loss function becomes:
\[
L_M(\theta) = \frac{1}{N} \| X\theta - y \|_2^2
\]
Even though the model includes powers of $x$, polynomial regression can still be solved using ordinary least squares because the model is linear in its parameters.

\subsection*{2.6 Accuracy vs Cross-Entropy Loss}
We assume that we have one sample:
\begin{itemize}
    \item True label: $y = 1$
    \item Model predicts $p = 0.6$ for class 1
\end{itemize}
That means accuracy on this sample = 100\% (model picks class 1).

Then we calculate the cross-entropy loss depending on the formula:
\[
L = -\left[1 \cdot \log(0.6) + 0 \cdot \log(0.4)\right]
   = -\log(0.6)
   = 0.51
\]
So accuracy = 100\%, but loss = 0.51 $>$ 0.

\subsection*{3.7 Comments on Learning Rate Decay}
\begin{itemize}
    \item The Rastrigin function is full of valleys and local minimum values.
    \item At the beginning, we need large increments to search quickly.
    \item When approaching the true minimum $(0,0)$, we need small increments to avoid overshooting.
    \item If the learning increment is constant, the algorithm may oscillate or converge very slowly.
    \item Therefore, \texttt{lr\_decay} helps stabilize convergence.
\end{itemize}
